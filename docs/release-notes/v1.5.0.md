# Waza v1.5.0

## 🎨 Codex-inspired UI redesign + Light/Dark theme

Complete UI overhaul matching OpenAI Codex app layout and aesthetics, with full light/dark theme support and local LLM integration.

---

## ✨ New features

### Theme system
- **Light / Dark theme** — Auto-detects OS setting (`prefers-color-scheme`). Toggle anytime with the button in the title bar.
- **Design token system** — `lightTokens` / `darkTokens` / `staticTokens` split for clean theme switching across all components.
- **Monaco editor sync** — Editor theme automatically switches between `vs` and `vs-dark`.

### Layout (Codex-aligned)
- **Sidebar** — New thread ✦ / Automations ◈ / Skills ⬡ navigation with SVG icons + thread group list + file tree fallback.
- **WelcomeScreen** — "Let's build" screen with 技 logo and folder selector when no file is open.
- **StatusBar** — Shows `Local | Worktree | Cloud` mode, Git branch (⎇), and cocoro-OS real-time connection status (🟢/🔴).
- **TitleBar** — macOS-style traffic lights, thread name, theme toggle, and updater badge.

### Composer
- **Inline model picker dropdown** — Select from `auto` / `cocoro-OS` / `Ollama` / `Claude Sonnet` / `Claude Opus` / `Gemini 2.0 Flash` directly in the compose area.
- Keyboard shortcut: `Enter` to send, `Shift+Enter` for newline.

### Local model support (cocoro-llm-server)
- `LocalProvider` — Connects to any OpenAI-compatible inference server.
- `isAvailable()` health check via `GET /v1/models` (3s timeout).
- Full SSE streaming support.
- `ModelRouter.routeWithPreference()` — Routes to local first, falls back to cloud on failure.

### Codex Mode (autonomous agent)
- `AgentPanelV2` — Live task status: Pause / Resume / Cancel / Retry controls.
- `ReviewModal` — Diff-based review gate before destructive file operations.
- `SkillsPanel` — Built-in skills (json-only / concise / test-first) + custom skill creation.
- `TelemetryPanel` — Provider health monitoring with 30s heartbeat.

## ⚙️ Configuration (local model)

```json
// .vscode/settings.json
{
  "waza.preferLocalModel": true,
  "waza.localModelUrl": "http://your-server:8000/v1",
  "waza.localModel": "your-model-name",
  "waza.localApiKey": ""
}
```

Set `WAZA_LOCAL_API_KEY` environment variable for the API key (takes priority over settings).

> **Note**: `.vscode/settings.json` is gitignored. Copy from `.vscode/settings.json.example`.

## 📦 Install

### VSCode Extension
```bash
code --install-extension waza-1.5.0.vsix
```

### Desktop app (Debian / Ubuntu)
```bash
sudo dpkg -i waza_1.5.0_amd64.deb
waza
```

### Requirements
- VSCode 1.87.0+
- Node.js 20+
- (Optional) Local model server: OpenAI-compatible API (`/v1/chat/completions`)
